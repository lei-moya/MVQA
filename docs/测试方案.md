# 短视频质量评级系统测试方案

> 本文档对应仓库内 **「视频质量分析平台」** 前后端实现，路径：`docs/测试方案.md`（由原 `testme.md` 整理入库）。内容应与 `README.md`、`frontend/src/components/Help.vue` 及 `backend/constants.py` 保持一致；若不一致以代码为准。

## 测试目标

1. 验证系统各功能模块正常运行
2. 确保前端和后端交互正常
3. 测试边界情况和异常处理
4. 验证性能和用户体验

## 测试环境

- **前端**：Vue 3 + Vite 7 + Element Plus + ECharts + Vue Router（开发态默认代理 `/api`、`/uploads`）
- **后端**：FastAPI + Uvicorn + SQLAlchemy 2.x + Python **3.9+**
- **数据库**：SQLite，库文件位于**项目根目录** `video_rating.db`（与进程 cwd 无关）
- **上传目录**：`backend/uploads/`（媒体与缩略图）
- **浏览器**：Chrome、Firefox、Edge（播放建议 H.264 + AAC 的 MP4）

## 测试用例

### 1. 前端测试

#### 1.1 文件上传功能

| 测试用例 | 测试步骤 | 预期结果 |
|---------|---------|--------|
| 单个视频文件上传 | 1. 点击"选择文件"按钮<br>2. 选择一个MP4视频文件<br>3. 点击"开始上传"按钮 | 文件上传按钮显示可用；文件上传成功，显示上传进度，分析完成后显示分析结果 |
| 仅弹幕无视频 | 1. 仅选择 ASS/XML 弹幕、未选视频<br>2. 尝试提交 | 前端或接口提示缺少视频或无法建任务（与 `FileUpload` 校验一致） |
| 视频+弹幕文件上传 | 1. 点击"选择文件"按钮<br>2. 选择一个MP4视频文件和对应的ASS弹幕文件<br>3. 点击"开始上传"按钮 | 文件上传成功，显示上传进度，分析完成后显示分析结果 |
| 少视频+多弹幕文件上传 | 1. 点击"选择文件"按钮<br>2. 选择一个MP4视频文件和多个ASS弹幕文件<br>3. 点击"开始上传"按钮 | 文件上传按钮显示可用；成对文件组上传成功，显示上传进度，分析完成后显示分析结果；未成对文件组上传失败，显示错误提示 |
| 多视频+少弹幕文件上传 | 1. 点击"选择文件"按钮<br>2. 选择一个MP4视频文件和多个ASS弹幕文件<br>3. 点击"开始上传"按钮 | 文件上传按钮显示可用；成对文件组上传成功，显示上传进度，分析完成后显示分析结果；未成对文件组上传失败，显示错误提示 |
| B 站 URL | 1. 在 URL 框粘贴有效 B 站稿件链接<br>2. 点击解析/提交 | 任务创建，落盘后进入分析；列表状态最终为 completed 或 failed |
| 非 B 站/无效 URL | 1. 粘贴非支持域名或无效链接<br>2. 提交 | 下载或解析失败，列表 failed 或接口/界面错误提示 |
| 批量本地上传 | 1. 使用「批量」入口仅多选视频（≤10 个）<br>2. 上传 | `BATCH_UPLOAD_MAX_FILES=10`，每条任务独立、无共享弹幕 |
| 文件格式验证 | 1. 尝试上传非允许扩展名<br>2. 观察提示 | 视频仅允许 `.mp4/.avi/.mov/.wmv/.flv/.mkv`；弹幕允许 `.ass/.xml`（与 `constants.py` 一致） |
| 文件大小限制 | 1. 尝试上传超过2GB的文件<br>2. 观察系统提示 | 系统显示文件大小超限提示 |
| 单组文件数量 | 1. 同一批选择超过 3 个视频或超过 3 个弹幕，或合计超过 6 个文件<br>2. 提交 | 前端提示超出单组上限（与帮助页「单组最多 3 视频 + 3 弹幕」一致） |

#### 1.3 交互功能

| 测试用例 | 测试步骤 | 预期结果 |

### 1.4 登录、设置与帮助

| 测试用例 | 测试步骤 | 预期结果 |

### 2. 后端测试

#### 2.1 API测试

| 测试用例 | 测试步骤 | 预期结果 |
|---------|---------|--------|
| 视频上传 API | 1. `POST /api/videos/upload`（multipart，需 Bearer）<br>2. 携带视频与可选弹幕 | 200，返回 `VideoResponse` |
| 批量上传 API | 1. `POST /api/videos/upload/batch` 多文件 | 200，返回视频数组，数量 ≤10 |
| URL 上传 API | 1. `POST /api/videos/upload/url`（表单/JSON 依实现） | 200，创建任务并开始下载/分析链路 |
| 获取视频详情 | 1. `GET /api/videos/{video_id}` | 200，详情 JSON |
| 获取视频列表 | 1. `GET /api/videos?skip=0&limit=30`（需 Bearer） | 200，`{ items, total, skip, limit }` |
| 重分析 API | 1. `POST /api/videos/{video_id}/reanalyze` | 200，任务重新排队 |
| 流播放 | 1. `GET /api/videos/{video_id}/stream`（含 Range） | 200/206，视频流；S3 场景可能 302 |
| 删除视频 API | 1. `DELETE /api/videos/{video_id}` | 200，记录与关联文件清理 |
| 健康检查 | 1. `GET /api/health` | 200，`{"status":"ok"}` 等 |
| 配置 API | 1. `GET/PUT /api/config`（PUT 需登录） | 与设置页一致 |

#### 2.2 业务逻辑测试

| 测试用例 | 测试步骤 | 预期结果 |
|---------|---------|--------|
| 视频分析 | 1. 上传视频<br>2. 等待分析完成<br>3. 检查分析结果 | 分析结果包含视频评分、片段评分和建议 |
| 同名文件处理 | 1. 上传一个视频文件<br>2. 再次上传同名视频文件 | 系统覆盖原文件，重新分析 |
| 异常处理 | 1. 上传损坏的视频文件<br>2. 观察系统处理 | 系统捕获异常，返回错误信息 |
| 后台任务 | 1. 上传多个视频<br>2. 观察后台处理情况 | FastAPI `BackgroundTasks` 异步执行分析，接口先返回；不依赖 Redis/Celery |

## 性能测试指标

| 测试指标 | 预期值 | 测试方法 |
|---------|-------|---------|
| 视频上传响应时间 | < 5秒 | 上传100MB视频文件 |
| 分析完成时间 | 依 GPU/CPU、分辨率与片段数而定 | 长视频或高 `num_clips` 可能达数分钟级 |
| API响应时间 | < 1秒 | 发送API请求 |
| 并发处理能力 | 支持5个并发上传 | 同时上传5个视频 |
| CPU使用率 | < 80% | 监控系统运行时CPU使用情况 |
| 内存使用率 | < 70% | 监控系统运行时内存使用情况 |

## 改进建议

| 问题描述 | 影响范围 | 建议解决方案 | 优先级 |
|---------|---------|------------|--------|
| 大文件上传速度较慢 | 用户体验 | 实现分片上传功能 | 中 |
| 分析完成后通知机制不完善 | 用户体验 | 添加邮件或系统通知功能 | 低 |
| 错误提示信息不够详细 | 用户体验 | 提供更具体的错误原因和解决建议 | 低 |
| 系统资源占用较高 | 性能 | 优化视频分析算法，减少资源消耗 | 中 |
| 安全性验证不足 | 安全 | 加强输入验证和文件类型检查 | 高 |
